How the observatory works

01

Log the promise

A scheduled GitHub Actions job polls the agencies’ public GTFS-realtime feeds in bursts, recording every “arriving in x minutes” with its timestamp.

02

Infer the truth

Feeds never announce arrivals, so the arrival is inferred: a prediction tracked to short range that then disappears from consecutive snapshots. Every inferred arrival carries an uncertainty, and only tight ones (≤120s) are graded — coverage is reported, not assumed.

03

Publish the record

Promise-vs-reality pairs accumulate into calibration curves and report cards on this page, rebuilt after every poll. The raw method, tests, and data are public.
